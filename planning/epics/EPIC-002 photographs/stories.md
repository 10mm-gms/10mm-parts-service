# EPIC-002 Photographs

## User Stories

### US-001 Removal of Image URL property

As a user, I want to remove the Image URL property from the part details modal so that I can store photographs in private S3-compatible storage (DigitalOcean Spaces).

#### Acceptance Criteria

- [ ] The Image URL property must be removed from the part details modal.
- [ ] The DB schema must be updated to remove the Image URL property.

### US-002 Upload Existing Photographs

As a user, I want to be able to upload photographs already exsisting on a user's device to a part. This should be done in the part details modal. It should be possible to upload multiple photographs for a part. It should be possible to mark a photograph as the primary photograph for a part.

#### Acceptance Criteria

- [ ] The photographs must be stored in private S3-compatible storage (DigitalOcean Spaces).
- [ ] The photographs must be uploaded directly to S3-compatible storage (DigitalOcean Spaces).
- [ ] When displayed on the client device, the photographs must be retrieved directly from S3-compatible storage (DigitalOcean Spaces).
- [ ] A maximum file size of 10Mb for any one photograph must be enforced.
- [ ] A maximum number of 30 photographs per part must be enforced.
- [ ] The original file name must be preserved, and when a photograph is downloaded, it must be downloaded with the original file name.
- [ ] Access to photographs must be restricted to authorized users via short-lived Presigned URLs generated by the backend.
- [ ] Immediately after upload, the photographs should be displayed in a user-friendly manner, such as in a modal or a lightbox. Photos should display as soon as they are uploaded, not after a page refresh, and not after all photos have been uploaded.
- [ ] File names should not need to be unique. The backend should generate a unique file name for each photograph, not forgetting the need to preserve the original file name for download purposes.
- [ ] File names should be checked for SQL injection vulnerabilities.
- [ ] **Client-side optimization**: Photos must be resized to a maximum of 2000px and converted to **WebP format** (approx. 75% quality) in the browser before upload to maximize compression efficiency.
- [ ] **Technical Validation**: Only standard image formats (JPEG, PNG, WEBP, HEIC) are permitted for selection. Non-image files or unsupported formats must be rejected by the client-side validation logic.
- [ ] **Metadata stripping**: Sensitive EXIF data (GPS coordinates, device info) must be stripped during the client-side optimization process.
- [ ] **Backend Signature Validation**: The backend must enforce strictly typed `Content-Type` (image/webp) and `Content-Length` restrictions in the presigned upload policy.
- [ ] **Post-Upload Verification**: After upload, the backend must verify the file header (Magic Bytes) to confirm it is a valid WebP image before finalizing the database record.

### US-003 Manage Photographs

As a user, I want to be able to manage photographs for parts and vehicles so that I can have a visual reference for the parts and vehicles in my inventory. For simplicity, the modal used to delete photographs should be the same modal used to upload photographs.

#### Acceptance Criteria

- [ ] The photographs should be deletable, such that authorized users can update or remove photographs as needed. There should be one delete button per photograph. There should be a confirmation dialog before a photograph is deleted.
- [ ] It should be possible to mark a photograph as the primary photograph for a part.

### US-004 Rapid-Fire Camera Capture

As a user, I want to be able to use my device's camera to take multiple photographs in rapid succession without leaving the camera interface, so that I can quickly document parts during an inspection.

#### Acceptance Criteria

- [ ] There must be a "Take Photo" mode that launches a live camera viewfinder within the application (using the MediaDevices API).
- [ ] It must be possible to snap photos repeatedly using a single shutter button.
- [ ] A visible counter must show the number of photos captured in the current session.
- [ ] The user must be able to switch between the front and rear cameras (if available).
- [ ] All security and optimization rules from US-002 (WebP, resizing, S3 direct upload, magic byte verification) must apply to these photos.
- [ ] The user should be able to review captured thumbnails before finalized upload.

## Dependencies

- [ ] EPIC-001 Parts and Vehicles

## Technical Implementation Notes

- Use the **Canvas API** on the client-side to implement the 2000px resize and **WebP** conversion/compression.
- S3 keys should be stored as UUIDs to ensure uniqueness and security.
- The `original_filename` must be stored as metadata in S3 and in the database.
- Use the `ResponseContentDisposition` parameter when generating download URLs to restore the original filename for the user.
- Enforce strict mimetype checking on the client before processing to prevent non-image uploads.
- The backend should perform an asynchronous check of the first 12 bytes of newly uploaded files to verify the "WebP" magic numbers before making them visible in the UI.
- Use the **MediaDevices API** for US-004 to build the custom viewfinder and frame-capture logic.

### US-005 View existing photographs

As a user, I want to be able to view photographs for parts and vehicles so that I can have a visual reference for the parts and vehicles in my inventory. The photographs should be displayed in a user-friendly manner, such as in a modal or a lightbox. 

#### Acceptance Criteria

- [ ] WHen I click on the primary photograph for a part, it should open in a modal or a lightbox. Only the primary photograph should be displayed. THere should be controls to navigate to the next and previous photographs
- [ ] I should be able to zoom in and out of the photograph
- [ ] I should be able to pan around the photograph
- [ ] I should be able to close the modal or lightbox by clicking on a close button or by pressing the escape key.
- [ ] I should be able to download the photograph by clicking on a download button.
- [ ] The individual part view modal should display a thumbnail of the primary photograph.
- [ ] The vehicle view modal should display a thumbnail of the primary photograph.
- [ ] The part list view should not display photographs.
- [ ] The vehicle list view should not display photographs.
